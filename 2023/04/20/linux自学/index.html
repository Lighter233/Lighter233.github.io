当前目录占204个字节（以k为单位）。

  1. 第一列显示文件的类型及文件的权限
  2. 第二列是文件的链接数
  3. 文件的属主
  4. 文件的组用户

  6. 后面三列是文件的最后修改时间

Linux grep (global regular expression) 命令用于查找文件里符合条件的字符串或正则表达式。

grep 指令用于查找内容包含指定的范本样式的文件，如果发现某文件的内容符合所指定的范本样式，预设 grep 指令会把含有范本样式的那一列显示出来。若不指定任何文件名称，或是所给予的文件名为 -，则 grep 指令会从标准输入设备读取数据。

进程操作

显示当前系统内的进程

Linux ps （英文全拼：process status）命令用于显示当前进程的状态，类似于 windows 的任务管理器。

语法

参数：

  • ps 的参数非常多, 在此仅列出几个常用的参数并大略介绍含义

  • -A 列出所有的进程

  • -w 显示加宽可以显示较多的资讯

  • -au 显示较详细的资讯

  • -aux 显示所有包含其他使用者的进程

  • au(x) 输出格式 :

    • USER: 行程拥有者
    • PID: pid
    • %CPU: 占用的 CPU 使用率
    • %MEM: 占用的记忆体使用率
    • VSZ: 占用的虚拟记忆体大小
    • RSS: 占用的记忆体大小
    • TTY: 终端的次要装置号码 (minor device number of tty)
    • STAT: 该行程的状态:
      • D: 无法中断的休眠状态 (通常 IO 的进程)
      • R: 正在执行中
      • S: 静止状态
      • T: 暂停执行
      • Z: 不存在但暂时无法消除
      • W: 没有足够的记忆体分页可分配
      • <: 高优先序的行程
      • N: 低优先序的行程
      • L: 有记忆体分页分配并锁在记忆体内 (实时系统或捱A I/O)
    • START: 行程开始时间
    • TIME: 执行的时间
    • COMMAND:所执行的指令

xargs 命令

xargs（英文全拼： eXtended ARGuments）是给命令传递参数的一个过滤器，也是组合多个命令的一个工具。

xargs 可以将管道或标准输入（stdin）数据转换成命令行参数，也能够从文件的输出中读取数据。

xargs 也可以将单行或多行文本输入转换为其他格式，例如多行变单行，单行变多行。

xargs 默认的命令是 echo，这意味着通过管道传递给 xargs 的输入将会包含换行和空白，不过通过 xargs 的处理，换行和空白将被空格取代。

xargs 是一个强有力的命令，它能够捕获一个命令的输出，然后传递给另外一个命令。

之所以能用到这个命令，关键是由于很多命令不支持|管道来传递参数，而日常工作中有有这个必要，所以就有了 xargs 命令，例如：

Init命令

在Linux中，init 0~6是七个系统运行级别（runlevel），每个运行级别对应不同的系统状态和服务集合。不同的运行级别可以用来启动或停止特定的系统服务或程序，以满足不同的系统运行需求。

具体解释如下：

  • 运行级别0（halt）：用于停止系统。执行init 0命令将会安全地关闭系统，并将所有服务和进程停止。
  • 运行级别1（single-user mode）：用于单用户模式。执行init 1命令将会启动单用户模式，即只有root用户能够登录，所有非必要的系统服务将会停止运行，适用于系统故障修复等操作。
  • 运行级别2（multi-user mode without NFS）：用于多用户模式，但不包括NFS网络文件系统服务。执行init 2命令将会启动多用户模式，并启动大多数系统服务，但不包括NFS网络文件系统服务。
  • 运行级别3（multi-user mode with NFS）：用于多用户模式，包括NFS网络文件系统服务。执行init 3命令将会启动多用户模式，并启动所有系统服务，包括NFS网络文件系统服务。
  • 运行级别4（unused）：这个级别没有特定的用途，可以根据需要自定义使用。
  • 运行级别5（graphical mode）：用于启动带有图形用户界面（GUI）的多用户模式。执行init 5命令将会启动多用户模式，并自动启动X Window系统和GUI桌面环境。
  • 运行级别6（reboot）：用于重启系统。执行init 6命令将会安全地重启系统，并重新启动所有服务和进程。

需要注意的是，在最新的Linux系统中，已经不再使用传统的init进程来管理系统启动和停止，而是使用更为先进的init系统（如systemd）。因此，这些运行级别的含义和使用方式可能会有所不同。

搜索替换

htons函数的作用是什么？

htons函数是一个网络字节序转换函数，其作用是将一个16位无符号短整型数值从主机字节序（host byte order）转换为网络字节序（network byte order）。

在计算机网络中，不同的计算机架构使用不同的字节序（byte order）来表示数据。主机字节序通常是指计算机处理器本身使用的字节序，而网络字节序是一种标准化的字节序，用于在不同计算机之间传输数据。

htons函数可以确保数据在网络上传输时采用正确的字节序，以便接收方能正确地解释数据。在发送数据时，通常会使用htons函数将数据转换为网络字节序；在接收数据时，通常会使用ntohs函数将数据从网络字节序转换回主机字节序。
